# TG WS Proxy

**TG WS Proxy** — локальный прокси MTProto для Telegram Desktop, использующий WebSocket в качестве транспортного уровня. Такой подход позволяет организовать стабильное соединение с серверами Telegram без применения сторонних MTProto-прокси, сохранив стандартный механизм шифрования протокола.

# Установка в Windows

Откройте страницу **Releases** и загрузите сборку, соответствующую вашей платформе.

Доступны следующие версии:

- `TgWsProxy_windows.exe` — Windows 10 и новее (x64);
- `TgWsProxy_windows_arm64.exe` — Windows 10 и новее (ARM64);
- `TgWsProxy_windows_7_64bit.exe` — Windows 7 (x64);
- `TgWsProxy_windows_7_32bit.exe` — Windows 7 (x86).

Во время первого запуска программа автоматически откроет инструкцию по настройке Telegram Desktop. После завершения конфигурации приложение продолжит работать в фоновом режиме и будет доступно через значок в системном трее.

## Возможности меню

Все основные функции доступны через контекстное меню значка приложения.

Доступные действия:

- **Открыть в Telegram** — сформировать ссылку `tg://proxy` и автоматически передать её клиенту Telegram.
- **Скопировать ссылку** — сохранить ссылку подключения в буфер обмена.
- **Перезапустить прокси** — перезапустить локальный MTProto-прокси без закрытия программы.
- **Настройки...** — открыть окно конфигурации и изменить параметры приложения, включая проверку обновлений.
- **Открыть журнал** — просмотреть журнал работы программы.
- **Выход** — завершить работу TG WS Proxy.

# Подключение Telegram Desktop

## Автоматическая настройка

Наиболее удобный вариант — воспользоваться автоматическим подключением.

Для этого:

1. Нажмите правой кнопкой мыши по значку TG WS Proxy в системном трее.
2. Выберите пункт **«Открыть в Telegram»**.
3. Подтвердите подключение прокси.

Если Telegram не обработал ссылку автоматически:

1. Откройте меню программы.
2. Выберите **«Скопировать ссылку»**.
3. Отправьте её в чат **«Избранное»**.
4. Перейдите по отправленной ссылке.
5. Подтвердите применение настроек.

## Ручная настройка

Если воспользоваться автоматическим способом невозможно, создайте подключение вручную.

Откройте раздел:

```text
Telegram
→ Настройки
→ Продвинутые настройки
→ Тип подключения
→ Прокси
```

Добавьте новое MTProto-подключение и заполните поля следующим образом:

| Параметр | Значение |
|----------|----------|
| Тип | MTProto |
| Сервер | `127.0.0.1` (либо адрес, указанный в настройках) |
| Порт | `1443` (или пользовательское значение) |
| Secret | секретный ключ из конфигурации приложения или журнала логов |
# Как устроена работа приложения

```text
Telegram Desktop
        │
        ▼
Локальный MTProto Proxy
        │
        ▼
WebSocket (TLS)
        │
        ▼
Telegram Data Center
```

После запуска TG WS Proxy принимает соединения от Telegram Desktop и выполняет роль промежуточного узла, перенаправляя весь сетевой трафик к серверам Telegram через защищённый WebSocket-канал.

Основной процесс выглядит следующим образом:

1. Запускается локальный MTProto-прокси с параметрами, указанными в конфигурации.
2. Принимаются подключения от Telegram Desktop.
3. Выполняется анализ **MTProto Obfuscation Init Packet**, после чего определяется идентификатор нужного дата-центра (DC ID).
4. Для выбранного сервера Telegram создаётся защищённое WebSocket-соединение.
5. Если подключение через WebSocket установить невозможно (например, сервер отвечает кодом **HTTP 302 Redirect**), приложение автоматически переключается на альтернативный маршрут — Cloudflare Proxy или прямое TCP-соединение.

> [!IMPORTANT]
>
> ### Медиафайлы перестали загружаться?
>
> Если после включения прокси Telegram перестал открывать фотографии, видео или другие медиафайлы, в первую очередь проверьте настройки списка IP-адресов дата-центров.
>
> Перейдите в раздел:
>
> **DC → IP**
>
> и оставьте только следующую запись:
>
> ```text
> 4:149.154.167.220
> ```
>
> Если проблема сохраняется, очистите содержимое этого поля полностью и повторите попытку подключения.
>
> Подобная ситуация может встречаться у пользователей без активной подписки **Telegram Premium**.
>
> Если указанные рекомендации не помогли, настройте собственный Cloudflare Proxy, воспользовавшись инструкцией:
>
> **[CfProxy.md](./CfProxy.md)**

# Автоматическая сборка

Проект уже включает все необходимые файлы, позволяющие автоматически собирать исполняемые версии приложения.

Для упаковки используются конфигурации **PyInstaller**, а процесс сборки и публикации автоматизирован через **GitHub Actions**.

Используемые спецификации:

- `packaging/windows.spec`
- `packaging/macos.spec`
- `packaging/linux.spec`

Workflow сборки расположен по адресу:

- `.github/workflows/build.yml`

Это позволяет формировать актуальные сборки автоматически после внесения изменений в репозиторий.
